- Исправлено ложное совпадение регионов в `matchRegion()` (двусторонний `indexOf` → нормализация + точное сравнение).

## v1.3.0 - 2026-07-07

Перестройка пайплайна публикаций (диагноз: `docs/agents/audit-2026-07-07.md`, редполитика: `docs/npz-posting-style-v2.md`).

- Добавлен `hermes/bot/render.py` — единый рендер-модуль (сводка/молния/апдейт/БПЛА-алерт) по редполитике v2: HTML-экранирование, entity-aware длина, автовыбор caption/split-mode, preflight-валидатор (allowlist тегов, без голых URL/пустых блоков). `python3 -m hermes.bot.render --selftest` зелёный.
- Добавлен `hermes/bot/day_state.py` — единое состояние дня + единый ключ дедупа `{date}:{тип}:{город}:{объект}` для всех форматов. Мигрирует старые `pipeline-state.json` / `~/.npz-bot/state.json` / `editorial-state.json` в общий `published_keys`.
- `broadcast.py`: `--briefing morning|evening` теперь рендерит через `render.py` и пишет state model; новый режим `--update` (пополнение сводки, editMessageCaption/Text, максимум 3 строки, авто-переключение caption→split при переполнении). Legacy diff/editorial-дайджест оставлены под `NPZ_LEGACY=1`, из publish-путей убраны.
- Молния (`strike_pipeline.py` + `radar_publish.py`) переведена из dry-run-only в прод: TIER1 — автопубликация в канал + подписчикам; TIER2 — владельцу с кнопками «✅ Опубликовать» / «🚫 Отклонить» (обработка в `poll.py`, который теперь также слушает `callback_query`). Единый дедуп на молнию/сводку.
- **Исправлен баг**: `strike_pipeline.py --dry-run` персистил `pipeline-state.json` в реальности (из-за `map_updated or not dry_run`), из-за чего повторные dry-run прогоны тихо "съедали" удары из очереди публикации. Теперь dry-run никогда не пишет state.
- Добавлен `strike_pipeline.py --backfill` — одноразовая пометка исторического беклога (65 ударов, накопленных за месяцы dry-run-режима) как обработанных без публикации, чтобы включение молнии в прод не устроило спам-рассылку по историческим событиям.
- `hermes/publish-vps.sh`: убран вызов `NPZ_EDITORIAL=1 broadcast.py` из post-публикационного пути — редакционный дайджест больше не постит в канал самостоятельно.
- Расписание (референс `hermes/crontab.hermes`): сводки перенесены на 05:00/17:00 UTC (08:00/20:00 МСК); NEWSWATCH больше не публикует напрямую — только пишет `strikes.json`, публикацию делает `strike_pipeline.py`. См. `docs/agents/jobs-migration-2026-07-07.md` для точной миграции джобов `~/.hermes/cron/jobs.json`.
