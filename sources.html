Методология

Все данные на карте — OSINT-оценка, основанная исключительно на открытых источниках. Мы не используем закрытые или классифицированные данные. Информация агрегируется автоматически набором routine-агентов (по одному на тип данных) и перекрёстно проверяется по нескольким независимым источникам.

Каждое событие (удар, изменение статуса НПЗ, ограничение) сохраняется вместе со ссылкой на первоисточник. Если по теме нет подтверждённых новостей — данные не обновляются: агенты никогда не генерируют вымышленные события.

Конвейер обновления данных
1
Сбор
Веб-поиск на русском и английском по 4–6 запросам на агента
2
Фильтрация
Дедупликация, проверка по дата+город, отсечение старых записей
3
Запись
Обновление JSON-файлов со строгим сохранением схемы
4
Валидация
Автоматическая проверка JSON, откат при ошибке
5
Публикация
Git push → GitHub → карта читает data/ напрямую из raw
Как собираются данные

Данные собирают автоматические routine-агенты — по одному на тип данных, итого 9 регулярно обновляемых наборов данных — плюс служебные скрипты: проверка свежести (вотчдог), обновление координат АЗС из OpenStreetMap, подтверждение ударов и валидация АЗС. Рутина запускается каждые 6 часов (4 раза в сутки). Ниже — основные направления мониторинга:

STRIKES
→ data/strikes.json
Удары БПЛА и ракетами по российским городам и объектам. Координаты, тип, жертвы, цели. Хранит до 110 последних событий.
NPZ-STATUS
→ data/fuel-state.json (статусы)
Статус каждого НПЗ и экспортного терминала: operational / partial / down. Мощности, степень повреждений, дата изменения.
FUEL-MARKET
→ data/fuel-state.json (рынок)
Регионы дефицита, национальный баланс, экспортный бан, биржевые цены. Ограничения на АЗС и объёмы импорта.
HISTORY-CRIMEA
→ data/history-crimea.json
Хроника событий + отдельный блок по Крыму: талоны, маршруты, статус АЗС, сухопутный коридор.
ROADS
→ data/roads.json
Топливные артерии: трасса Р-280 «Новороссия», коридор Мариуполь, Чонгар. Удары по бензовозам, статус дорог.
ORCHESTRATOR
→ все data/*.json
Координирует остальных агентов, управляет git pull/push, валидирует JSON и откатывает битые файлы.

Важно: Все данные на карте являются оценкой на основе открытых источников (OSINT). Это не официальная статистика. Мощности НПЗ, объёмы выпуска и потери — аналитические оценки. Вкладка «АЗС»: расположение заправок реальное (OpenStreetMap), но наличие топлива — оценка по статусу сети в регионе, не данные по конкретной колонке. Маршрут показывает риск дефицита по регионам и не гарантирует наличие топлива на конкретной АЗС; «карта поездки» носит справочный характер. Агенты работают автоматически и могут ошибаться: при обнаружении неточности пишите через GitHub Issues проекта. Каждое событие в базе данных содержит ссылку на первоисточник — вы можете проверить любой факт самостоятельно.
