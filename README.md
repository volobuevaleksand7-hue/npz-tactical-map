# ТОПЛИВНЫЙ ФРОНТ РФ // NPZ TACTICAL MAP

Тактическая дашборд-карта состояния нефтепереработки России на фоне ударов по НПЗ.

🟢 работает · 🟡 частично · 🔴 выбит — по каждому НПЗ: мощность, оператор, статус, дата, характер повреждений, источник. Плюс слой логистики (нефте/продуктопроводы, экспортные терминалы) и регионы дефицита на АЗС. Слева — нац. баланс переработки и расклад «внутренний рынок ⟷ экспорт» по бензину и дизелю.

> ⚠ **Режим ОЦЕНКА.** Статусы и даты — из открытых новостей (OSINT). Объёмы и баланс — расчётная оценка по паспортным мощностям. Точных real-time данных по остаткам топлива публично не существует. Это не официальная и не разведывательная информация.

## Стек

Статический сайт (HTML/CSS/JS + Leaflet, dark-tiles CARTO, без API-ключей). Данные — единственный файл `data/fuel-state.json`, который фронт перечитывает каждые 5 минут (онлайн-режим).

```
index.html        # разметка + тактический HUD
styles.css        # «военный» стиль: скан-линии, скобки, моноширинный шрифт
app.js            # карта, маркеры, панели, тикер, авто-синхронизация
data/fuel-state.json  # единый источник данных (его обновляют агенты)
vercel.json       # заголовки (no-cache для data/)
agents/           # cron-агенты на VPS, держат данные свежими (см. agents/README.md)
```

## Как обновляются данные онлайн

Два дешёвых (Haiku) агента на VPS по cron парсят свежие новости → переписывают `data/fuel-state.json` → `git push` → Vercel авто-редеплоит → карта подтягивает обновление. Детали — [agents/README.md](agents/README.md).

## Локальный запуск

```bash
cd npz-tactical-map
python3 -m http.server 8080
# открыть http://localhost:8080
```

## Деплой

Статика на Vercel (framework = Other). Пуш в `main` = авто-деплой.
