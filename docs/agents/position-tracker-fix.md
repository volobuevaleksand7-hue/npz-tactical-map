# Position tracker: РЕШЕНО 15.07 — трекер удалён, причина устранена на VPS

## Итог

`data/yandex-positions.json` удалён из репо (вместе с `.old`,
`-apify.json`, `-last-success.json`). Он был мёртвым грузом: ноль
потребителей в коде/фронте, только сломанные данные чужого проекта.
NPZ-позиции в поиске меряются `agents/almost-there.py` — прямой запрос к
Яндекс.Вебмастер API (бесплатно, без капчи, без скрейпинга).

`data/position-queries.json` — оставлен как справочник (40 курируемых
NPZ-запросов с реальными частотами Wordstat и целевыми URL), но никакой
скрипт его больше не читает.

## Первопричина (найдена 15.07, а не то, что предполагалось 11.07)

Файл писала НЕ watchdog-рутина npz, а полностью посторонний cron-джоб
Гермеса (`~/.hermes/cron/jobs.json`, id `41c7f8787682`, ранее назывался
**«NPZ Yandex Positions»** — вот откуда путаница) для **чужого проекта
logika-itp** (логика-итп.рф). Джоб гоняет `*/30 5-20 * * 1-5` и напрямую
запускает `~/.hermes/scripts/yandex-position-check.py` (`no_agent: true`,
без LLM) — Playwright-скрейпер выдачи Яндекса по ключам «обслуживание
итп»/«обслуживание итп Москва». У скрипта (и у соседнего
`yandex-xmlriver-check.py`) был **захардкожен** путь вывода:
`/root/npz-tactical-map/data/yandex-positions.json`.

Коммит `a3187d7` (11.07) правил только содержимое файла в репо — не
трогал сам джоб. На следующем 30-минутном тике посторонний скрипт снова
перетёр файл ключами ИТП. Дальше файл утекал в git через
`agents/git-sync.sh`, который делает `git add data/` (весь каталог) —
любая радар-рутина (`cron-radar-refresh.sh`, каждые 10 минут) подхватывала
чужой грязный файл и коммитила его под своим сообщением
(`data(radar): refresh ...`), маскируя источник.

## Что исправлено на VPS (15.07)

1. `~/.hermes/scripts/yandex-position-check.py` и `yandex-xmlriver-check.py`:
   `OUTPUT_FILE`/`output_path` переключены на
   `/root/.hermes/data/logika-itp-positions.json` — вне репо npz-tactical-map.
   Проверено запуском вручную: пишет в новый файл, `data/yandex-positions.json`
   (тогда ещё существовавший) остался нетронут (md5 не изменился).
2. Джоб `41c7f8787682` в `~/.hermes/cron/jobs.json` переименован в
   **«ITP Yandex Positions (logika-itp, НЕ npz)»**, текст промпта обновлён
   на новый путь. Бэкап конфига: `~/.hermes/cron/jobs.json.bak-2026-07-15-tracker-fix`.
3. В репо: удалены `data/yandex-positions*.json` (весь семейство файлов,
   больше нечего — источника записи в репо не осталось физически).

## Почему не откатится в третий раз

Причина устранена у источника: скрипт, который раньше писал в этот путь,
больше не знает о нём вообще. Даже если джоб `logika-itp` продолжит
работать вечно — он пишет в `~/.hermes/data/`, вне git-дерева
npz-tactical-map. Файла-цели для случайного `git add data/` не существует.

Если понадобится живой трекер позиций именно NPZ — не воскрешать
скрейпер, звать `agents/almost-there.py` (уже в репо, работает).
