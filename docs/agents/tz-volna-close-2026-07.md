# ТЗ — Закрыть «Волну дронов»: боевой автопаблиш + хвосты — 2026-07-17

Один прогон. Цель — довести фичу «Волна дронов» из режима наблюдения в боевой
автопаблиш, закрыв хвосты, вскрытые при починке детектора 17.07.

## Зачем это сейчас (условие выполнено именно сегодня)

Фича раскатана 13.07 **в режиме наблюдения** (kill-switch `NPZ_WAVE_AUTOPUBLISH`).
Условие включения боевого режима было сформулировано как «после калибровки порога
над реальной волной». Калибровка сделана 17.07 (коммит `effb0808`):

- **Корень промаха:** не счёт 25/4, а окно свежести. В затяжной широкой волне
  город остаётся `bpla && !bplaDim` часами, но `last_event_ts` стареет за 45 мин
  и город выпадает из счёта. Реплей `evaluate()` по git-истории radar-state за
  ночь 16→17.07: пик одновременно-свежих держался **~20 при пороге RISE=25** →
  волна из 10 регионов не детектилась вовсе, хотя `bpla_notdim` был 40+.
- **Фикс:** `FRESH_SEC` 45→90 мин (свип по реальным снимкам: 45→пик 20 промах,
  90→пик 27 при дневном штиле ≤11, 180→штиль 26 = ложняк) + EMPTY-guard на
  пустой `cities[]`.
- **Проверка:** пропатченный детектор на всех снимках ночи даёт ровно одну
  чистую волну — start `2026-07-16T20:50:03Z` → end `21:50:03Z`, пик 27 городов
  / 9 регионов одновременно, 10 уникальных в накопленном списке. 6/6 тестов.

**Статус на 17.07:** `NPZ_WAVE_AUTOPUBLISH` в `~/.npz-agent.env` на VPS **не
задан** (проверено) → детектор считает и копит `wave-events.json`, но статей не
генерит. Фича 4 суток работает вхолостую.

## Задача 3 🟡 — Root cause: второй писатель `radar-state.json`

Вскрыто при починке, не копалось. **Это не косметика:** пустой снимок мог
ложно закрывать активную волну; сейчас прикрыто EMPTY-guard'ом, но причина жива.

### Факты
- `radar-state.json` пишут **два источника**:
  - основной — city-level, `cities[]` заполнен, `fetched_at` с микросекундами;
  - второй — region-схема: `cities[]` **пуст**, данные в `regions{}`,
    `fetched_at` целыми секундами, идёт ~50 с позже основного,
    `version:23`, `geo_parser_version:4`.
- **18 из 116** коммитнутых снимков за ночь 16→17.07 (~16%) — с пустым `cities[]`.
- Детектор читает только `cities[]`.

### Что сделать
1. Найти второго писателя (кто пишет `radar-state.json` в region-схеме и зачем).
   Смотреть `hermes/` cron-скрипты и агентов на VPS, а не только репо.
2. Ответить на вопрос: это **легитимный фолбэк** (upstream отдал только регионы)
   или **баг/гонка** (два писателя дерутся за один файл)?
3. По ответу — одно из:
   - легитимный фолбэк → оставить EMPTY-guard, задокументировать схему;
   - гонка/баг → развести писателей (разные файлы) или починить писателя.
4. 🔴 Не «чинить» вслепую: сначала доказать, кто пишет.

### Критерии приёмки
- Писатель назван поимённо (файл/крон/агент), причина пустого `cities[]` — тоже.
- Решение обосновано фактами, а не догадкой.
