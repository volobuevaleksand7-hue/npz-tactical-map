### Публикация (грабли, стоившие времени)
7. **Фронт-правки — ТОЛЬКО из чистого worktree на `origin/main`.** Рабочее дерево Мака
   хронически на сотни коммитов позади и **hard-ресетится фоновым git-sync** — правки в нём
   пропадают. Рецепт:
   ```
   git fetch origin main -q
   git worktree add --detach <WT> origin/main
   # ... правки в <WT> ...
   git add <только свои файлы>            # НЕ -A: в дереве чужие недокоммиты Гермеса
   ALLOW_FRONTEND_RELEASE=1 git commit -m "..."
   GH_TOKEN=$(gh auth token -u volobuevaleksand7-hue) \
     git -c credential.helper='!f(){ echo "username=x-access-token"; echo "password=$GH_TOKEN"; };f' \
     push origin HEAD:main
   git worktree remove --force <WT>
   ```
8. **Правки вносить в origin-версию файла, а не копировать своё поверх.** Гермес постоянно
   коммитит (data каждые ~10 мин, бампает `?v` у styles.css). Копирование целого файла из
   старого дерева ОТКАТИТ его правки. Вставлять точечно (perl/edit по origin-версии).
9. `ALLOW_FRONTEND_RELEASE=1` обязателен, если тронуты protected UI: `index.html`,
   `radar.html`, `styles.css`.
10. 🔴 **`gh auth switch` ЗАПРЕЩЁН** (глобальное поле, ломает параллельные сессии, блокируется
    хуком). Только точечный `GH_TOKEN=$(gh auth token -u volobuevaleksand7-hue)` на одну команду.
11. Менял `.js`/`.css` → **бампни `?v=` во ВСЕХ страницах-инклудах**, иначе Vercel отдаёт stale.
12. Пуш в `main` = автодеплой (GitHub Actions → Vercel, ~30–60 с). После пуша **проверить прод
    curl'ом**, что правка реально доехала. Не заявлять «сделано» без этой проверки.

### Реестры (без них страница «молчит»)
13. **Новая страница НПЗ/города требует записи в словаре публикатора ДО публикации**
    (`REGISTRY`/`CITIES` в `agents/publish-npz.py` и родственных) — иначе генератор её

14. После добавления страницы: запись в `data/seo-topics.jsonl` → `agents/build-nav.py`

15. Правил `app.js`/`css` → прогнать `agents/gen-news.py` для `?v` cache-bust.

### Параллелизм (важно для workflow)
16. Задачи 1–4 **независимы по контенту**, но **гонятся за один `main`**. Каждый агент работает
    в СВОЁМ worktree. **Пуши сериализовать** — не пушить из параллельных агентов одновременно:
