# БРИФ: Оркестратор SEO (кто угодно с правом решать)

Исполнитель по умолчанию: Claude (Fable/Opus) в сессии с владельцем, понедельник.
Передача: Гермес-рутина — только после месяца стабильного KPI (PLAN.md шаг 5).

## Роль
Единственная роль, которая ПРИНИМАЕТ РЕШЕНИЯ: что реоптимизировать, какую тему
писать, что отклонить. Остальные роли только исполняют или готовят данные.

## Входы
1. `docs/agents/seo/reports/<текущая неделя>.md` — отчёт аналитика (если его нет — сначала запусти аналитика, без отчёта не решать).
2. `data/seo-backlog.jsonl` — бэклог тем от разведчика.
3. `data/seo-topics.jsonl` — реестр существующих страниц.
4. `docs/seo-playbook.md` — закон; при конфликте с чем угодно побеждает playbook.

## Шаги
1. Прочитай отчёт аналитика: возьми страницы на позициях 5–15 → это кандидаты на реопт (быстрый выигрыш).
2. Выбери 1–2 страницы на реопт → задача реоптимизатору (список: URL + что именно освежить).
3. Из бэклога выбери max 1 тему со status=idea: проверь против реестра (≥40% ключей покрыто → reject с note), частотность, тип. Годится → status=approved → задача писателю.
4. Отклонённые темы — status=rejected + причина в note. Бэклог не должен гнить.
5. Запиши решения недели одной строкой в `AGENT_ACTIVITY.md ## Log`.

## Выходы
- Обновлённые статусы в `data/seo-backlog.jsonl`.
- Задачи реоптимизатору и писателю (текстом в сессии или scheduled-task).
- Строка в `AGENT_ACTIVITY.md`.

## Запреты
- Не создавать страницы самому — только назначать.
- Не больше 1 новой страницы в назначении за цикл (playbook §6).
- Не решать без отчёта аналитика «по ощущениям».
