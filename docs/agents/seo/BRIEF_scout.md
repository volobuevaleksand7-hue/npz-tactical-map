# БРИФ: Разведчик семантики (открытые данные — делегируется свободно)

Исполнитель по умолчанию: Gemini CLI (`gemini -m gemini-3-pro-preview`, web-grounding),
фолбэк — Claude-субагент. Можно: DeepSeek-Flash, Гермес. Секретов в роли нет.

## Роль
Ищет новые поисковые кластеры и пополняет бэклог тем. Сам страниц НЕ создаёт
и решений НЕ принимает — только предлагает со статусом idea.

## Входы
1. Wordstat / поисковые подсказки Яндекс+Google / «люди также ищут» (открытые данные).
2. Раздел «Дыры» последнего отчёта аналитика `docs/agents/seo/reports/`.
3. `data/seo-topics.jsonl` — реестр: тема с ≥40% пересечением ключей с существующей страницей НЕ предлагается.
4. `data/seo-backlog.jsonl` — не дублировать уже предложенное (включая rejected: их не переподавать без нового аргумента в note).
5. Конкуренты выдачи (какие страницы стоят в топе по кластеру, чем слабы).

## Шаги
1. Возьми 2–3 направления: дыры из отчёта аналитика + смежные к живым кластерам сайта (топливо/АЗС/удары/радар/регионы).
2. Для каждого кандидата собери: главный ключ, 5–10 хвостов, оценку частотности, тип по playbook (region/object/explainer/forecast/reference).
3. Прогони анти-каннибализацию против реестра (п.3 входов). Пересечение — отбрось.
4. Добавь прошедших в `data/seo-backlog.jsonl` по строке на тему, status=idea, source указать.
5. Итог — 3–7 тем за прогон, не больше: качество отбора важнее объёма.

## Выходы
- Новые строки в `data/seo-backlog.jsonl` (закоммитить по git-протоколу; исполнитель без git-доступа отдаёт строки JSONL текстом — коммитит оркестратор).

## Запреты
- Не создавать страницы, не менять реестр `seo-topics.jsonl`.
- Не предлагать темы вне тематики сайта и темы, требующие небезопасного контента (метки ПВО и т.п. — см. CLAUDE.md проекта).
- Частотность без источника не писать — лучше `freq_est: null`, чем выдумка.
