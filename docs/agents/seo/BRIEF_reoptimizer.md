# БРИФ: Реоптимизатор (свежесть)

Исполнитель по умолчанию: Claude scheduled-task / сессия. Передача: Гермес-крон
(шаг 5 PLAN.md) — роль почти механическая.

## Роль
Освежает СУЩЕСТВУЮЩИЕ страницы из списка аналитика: данные, даты, CTA,
перелинковка на новые страницы. Новых страниц не создаёт.

## Входы
1. Список «Кандидаты на реопт» из свежего отчёта `docs/agents/seo/reports/` (или прямое назначение оркестратора). Без списка — не работать.
2. `data/*.json` — свежие цифры/статусы для подстановки.
3. `docs/seo-playbook.md` §6 (ритм реоптимизации) и §1 (анатомия).

## Шаги
1. Для каждой назначенной страницы (1–2 за прогон): обнови цифры/статусы из `data/*.json`, дату «Обновлено <дата> МСК», `dateModified` в JSON-LD.
2. Добавь 1–2 внутренние ссылки на страницы, вышедшие после последнего обновления (сверься с `seo-topics.jsonl` по датам).
3. Проверь CTA: со статейных страниц должен вести на карту/радар; с региональных — на слой АЗС региона.
4. `lastmod` страницы в `sitemap.xml` → сегодня.
5. `python3 agents/check-ia.py` зелёный → коммит по git-протоколу HERMES.md §0 (чистый клон при грязном дереве), прод 200.

## Выходы
- Обновлённые страницы + sitemap, один логический коммит на прогон.
- Строка в `AGENT_ACTIVITY.md ## Log` (какие URL освежены).

## Запреты
- Не переписывать структуру/интент страницы — это решение оркестратора, не твоё. Только свежесть.
- Не трогать страницы вне назначенного списка.
- UI-shell не трогать. Русские даты, нейтральный тон — как у писателя.
