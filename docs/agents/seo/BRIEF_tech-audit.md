# БРИФ: Техаудитор

Исполнитель по умолчанию: детерминированные скрипты + любая дешёвая модель на
разбор (DeepSeek-Flash / Claude Haiku). Передача: Гермес-крон (шаг 5 PLAN.md).
Секретов нет, но и веб-аккаунты не нужны — всё по репо и живому сайту.

## Роль
Еженедельная техническая проверка. Находит поломки, НЕ чинит контент — чинит
только техничку (sitemap, canonical, битые ссылки) или отдаёт находку оркестратору.

## Чек-лист прогона (пятница)
1. `python3 agents/check-ia.py` — реестр/nav/страницы согласованы.
2. Sitemap: все URL из `seo-topics.jsonl` присутствуют, все отвечают 200 на проде (`curl -s -o /dev/null -w '%{http_code}'`), нет URL-сирот в sitemap без файла.
3. Canonical: каждая страница self-canonical на чистый URL без `.html`.
4. JSON-LD: все блоки парсятся `json.loads`; в Article `dateModified` не старше даты «Обновлено» на странице.
5. Внутренние ссылки: нет битых (каждая /ссылка есть в реестре или в корне репо).
6. Кеш-доставка: `sw.js`/`version.json` на проде отдаются с `max-age=0` (см. память об инциденте v1.10.3 — Vercel кешировал sw.js).
7. Индексация: случайные 3 страницы прошлой недели — `site:` проверка в Яндексе/Google (появились ли в индексе).

## Выходы
- Мелкая техничка (lastmod, битая ссылка, отсутствующий canonical) — починить сразу, коммит по git-протоколу HERMES.md §0.
- Всё контентное/структурное — списком в конец свежего отчёта аналитика (`## Техаудит` секция) или отдельной заметкой оркестратору.
- Строка в `AGENT_ACTIVITY.md ## Log`.

## Запреты
- Не менять контент страниц, реестр и бэклог.
- UI-shell не трогать (даже если находка в нём — только доложить).
