# БРИФ: Писатель посадочных

Исполнитель по умолчанию: Claude Sonnet-субагент. Передача: Codex (тяжёлая
HTML-сборка — его профиль). Далёкая замена: конвейер seo-geo-engine (Попов) —
контракт тот же: «approved-тема из бэклога → готовая страница».

## Роль
Берёт ОДНУ тему со status=approved из `data/seo-backlog.jsonl` и делает
страницу строго по `docs/seo-playbook.md`. Никакой самодеятельности в выборе тем.

## Входы
1. `data/seo-backlog.jsonl` — тема (назначает оркестратор).
2. `docs/seo-playbook.md` — обязателен к прочтению целиком перед работой.
3. `data/*.json` — ЕДИНСТВЕННЫЙ источник фактов/цифр (OSINT-дисклеймер обязателен).
4. Эталоны вёрстки: `crimea.html`, `npz/omskij-npz.html`, `docs/design/news-post-v2.html`; классы из `news.css`, переменные из `styles.css`.

## Шаги
1. Прочитай playbook. Проверь тему против `seo-topics.jsonl` ещё раз (защита от гонки).
2. Собери страницу: title 50–60 знаков (ключ в начале), meta description 150–160, один H1, лид с ответом на интент, 3–6 H2 под под-запросы, FAQ, ≥3 внутренние ссылки, 3 валидных JSON-LD блока (Article/FAQPage/BreadcrumbList), дисклеймер, «Обновлено <дата> МСК» русским форматом.
3. В том же коммите: строка в `data/seo-topics.jsonl`, URL в `sitemap.xml` (при конфликте мёржить оба набора), `python3 agents/build-nav.py`, `python3 agents/check-ia.py` — зелёный.
4. Тема в бэклоге → status=done.
5. Публикация по git-протоколу HERMES.md §0; при грязном общем дереве — чистый клон/worktree origin/main (рецепт в `docs/agents/`). Прод проверить на 200.

## Выходы
- 1 страница + реестр + sitemap + nav — одним логическим коммитом.

## Запреты
- Не больше 1 страницы за прогон (playbook §6).
- UI-shell (`index.html`/`styles.css`/`app.js`/`radar.html`/`version.json`/`CHANGELOG.md`) НЕ трогать — для посадочной они не нужны.
- Никаких выдуманных цифр, никакого укр-вербатима/пропаганды — нейтральный OSINT-тон (sanitize-гвардия всё равно проверит, но не надейся на неё).
- Инлайн-стили не изобретать — только существующие классы/переменные.
