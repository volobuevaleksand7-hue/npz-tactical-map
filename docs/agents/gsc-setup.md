# Google Search Console — инструкция владельцу

Статус на 2026-07-07: код верификации в `index.html:16` уже реальный
(`kawGMc_7WeiQ-fTGk3TcwM7luFAK7S_HZsd76OSpwlM`), вставлен одним из агентов.
Эта инструкция остаётся как справочник — на случай смены домена/аккаунта Google
или если верификацию нужно будет переделать.

## 1. Где взять meta-код верификации

1. Зайти на [search.google.com/search-console](https://search.google.com/search-console)
   под аккаунтом Google, которым владеет проект.
2. «Добавить ресурс» → выбрать **«Префикс URL»** (не «Домен» — префикс не требует
   доступа к DNS, только к файлам сайта): `https://npz-tactical-map.vercel.app/`
3. В способах подтверждения выбрать **«HTML-тег»** (самый простой для статического
   Vercel-сайта — DNS-запись недоступна, а «файл на сервере» требует лишнего деплоя).
4. Google покажет строку вида:
   ```html
   <meta name="google-site-verification" content="XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX" />
   ```
   Скопировать только значение `content="..."`.

## 2. Куда вставить

Файл `index.html`, строка 16:
```html
<meta name="google-site-verification" content="ВСТАВИТЬ_СЮДА_КОД">
```
Строка уже существует и закомментирована пояснением на строке 14-15 — просто
заменить значение `content`. Больше никаких файлов трогать не нужно: Google
проверяет только главную страницу домена.

**Важно:** `index.html` — защищённый UI-shell файл (см. `AGENT_ACTIVITY.md` и
`HERMES.md`). Коммит с этой правкой требует `ALLOW_FRONTEND_RELEASE=1` и записи
в `AGENT_ACTIVITY.md`, как и любая другая правка `index.html`.

## 3. Как подтвердить и отправить sitemap

1. После вставки кода и деплоя на Vercel — вернуться в Search Console и нажать
   «Подтвердить».
2. Слева в меню открыть **«Индексирование» → «Файлы Sitemap»**.
3. Ввести `sitemap.xml` (относительно корня — Search Console сама подставит
   `https://npz-tactical-map.vercel.app/sitemap.xml`) и нажать «Отправить».
4. Проверить статус через 1-2 дня: Search Console покажет число найденных и
   проиндексированных URL. Если ошибки — они появятся в разделе «Страницы».

## 4. Что мониторить дальше (рекомендация)

- **Эффективность → Запросы**: следить за позициями по ключам «нет бензина»,
  «в крыму нет бензина», «топливо в крыму», «дефицит бензина россия» —
  именно под них сделаны `/`, `/crimea`, `/deficit`.
- **Индексирование → Страницы**: убедиться, что `/crimea`, `/deficit`,
  `/npz/omskij-npz` и даты `/news/<date>` не попадают в «Исключено» —
  особенно после смены `cleanUrls: true` в `vercel.json` (проверить, что
  редиректов на `.html`-версии URL нет — Google не любит дублирующиеся
  канонические адреса).
- **Улучшения → Товары с разметкой / Результаты поиска**: проверить, что
  `FAQPage`, `Article`/`NewsArticle` и `BreadcrumbList` подхватываются без
  ошибок (обычно видно через 1-2 недели после подтверждения).
