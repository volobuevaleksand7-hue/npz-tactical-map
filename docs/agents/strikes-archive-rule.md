# ПРАВИЛО: data/strikes.json — полный архив, только дописывать

**Для всех коллекторов (Гермес, облачные рутины, ручные агенты).** Введено 2026-07-09 после второй регрессии.

## Что случилось (дважды)

- Июнь: коллектор отсёк записи старше окна → 168→110, утекли удары мая (вкл. Чебоксары).
- 08.07 23:20Z: коллектор перезаписал файл свежим снапшотом → 147→0→5. Восстановлено в v1.9.1 (148 записей).

## Правило

1. `data/strikes.json` — **полный архив с 06.06.2026**, не «окно за N дней». Витрины сами фильтруют (радар — 48ч, главная — пресеты).
2. Коллектор **читает существующий файл, дописывает новые записи (дедуп по `id`), никогда не удаляет старые**.
3. Схема записей: `lat`/`lon` + `detail` (старая) ИЛИ `location:[lat,lon]` + `description` (новая) — фронт понимает обе, но новые записи писать в **старой** схеме (`lat`, `lon`, `detail`), чтобы не плодить drift.
4. Удаление записей (чистка пропаганды/дублей) — только осознанно, с `ALLOW_STRIKES_SHRINK=1`.

## Страховка

Pre-commit hook (`.githooks/pre-commit`, включается `agents/git-sync.sh` через `core.hooksPath`) **блокирует коммит**, если число записей падает >10%:

```
pre-commit: BLOCKED — data/strikes.json усыхает: 148 → 4 записей.
```

Если словил блок — ты перезаписал архив: возьми `git show HEAD:data/strikes.json`, смержи свои новые записи в него и коммить снова.
