# План — Энергетический слой (ТЭС / ГЭС / АЭС / ТЭЦ) — 2026-06-22

Статус: **ПЛАН, не реализовано.** Реализация — отдельной сессией после правок по REVIEW_2026-06-22.

## Цель

Добавить на карту визуализацию объектов **генерации электроэнергии** РФ — ТЭС, ГЭС, АЭС, крупные ТЭЦ — с тем же визуальным качеством и логикой статусов, что у объектов НПЗ. Зачем: удары и аварии всё чаще задевают энергетику (подстанции уже есть в слое «⚡ Свет» = `grid-state.json`, но это **отключения**, а не **генерация**). Энергетический слой показывает сами электростанции: тип, мощность, оператор, статус, последний инцидент.

> Важно не путать с существующим: `⚡ Свет` (`L_ru.grid`, `data/grid-state.json`) = подстанции + блэкауты. Новый слой = **электростанции/генерация**.

## Решение по форме: отдельная вкладка «ЭНЕРГЕТИКА»

Зеркалим паттерн вкладки **АЗС** (`view-azs`): новый `<section class="view" id="view-energy">` со своей Leaflet-картой `#mapEnergy`, панелью-списком объектов, легендой и фильтром по типу. Качество отрисовки объектов — как у НПЗ (форма = тип объекта, цвет = статус, контур = выбран).

Альтернатива (можно добавить позже, не вместо): тоггл-слой `data-layer="energy"` на карте РОССИЯ для совместного анализа «удары × НПЗ × энергетика» на одном полотне. Рекомендация: начать с вкладки, слой-оверлей — фаза 3.

## Модель данных — `data/energy.json`

```json
{
  "generated_at": "2026-06-22T03:00Z",
  "source": "OSINT estimate (OSM + Wikipedia + операторы + СМИ)",
  "plants": [
    {
      "id": "sayano-shushenskaya",
      "name": "Саяно-Шушенская ГЭС",
      "type": "hydro",                // hydro | thermal | nuclear | chp
      "lat": 52.829, "lon": 91.371,
      "capacity_mw": 6400,
      "operator": "РусГидро",
      "region": "Хакасия",
      "status": "operational",        // operational | reduced | damaged | offline | unknown
      "last_event": null,             // {date, type:"strike|accident|maintenance", note, src}
      "note": "Крупнейшая ГЭС России",
      "src": "https://..."
    }
  ]
}
```

Типы и визуальная грамматика (review #8):
- `thermal` ТЭС — квадрат, 🔥/дым-иконка
- `hydro` ГЭС — капля/плотина
- `nuclear` АЭС — гекс/⚛
- `chp` ТЭЦ — квадрат с трубой
- Цвет статуса: `operational` зелёный · `reduced` жёлтый · `damaged` оранжевый · `offline` красный · `unknown` серый.

## Риски / решения
- **Путаница с «⚡ Свет».** Решение: разные названия/легенды; вкладка = генерация, слой Свет = отключения. Возможно переименовать тоггл в «⚡ Сети».
- **Достоверность статусов.** По умолчанию `operational`; `damaged` только при подтверждённом источнике (как со strike-confirm).
- **Плотность маркеров** (каскад ГЭС) → кластеризация обязательна.
- **Чувствительность темы.** Это OSINT по публичной инфраструктуре; держать дисклеймер «оценка», без оперативных целеуказаний.

## Оценка
Фаза 0–1 (вкладка + сид-данные + рендер) — основной объём. Фаза 2–3 — инкремент. Точные данные сида лучше собрать отдельным research-проходом (можно делегировать DeepSeek/MiMo на открытых источниках).
