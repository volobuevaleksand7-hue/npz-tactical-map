## Executive summary

The dashboard is visually strong and already reads as a serious OSINT war-room product. The top risks are not visual polish. They are trust, first-time comprehension, social distribution, and mobile discoverability.

Fix trust and onboarding before adding more map layers.

## P0 / P1 вАФ fix first

### 1. Broken social preview image

Evidence:
- `index.html` advertises `https://npz-tactical-map.vercel.app/og-image.png` for `og:image` and `twitter:image`.
- Live URL returns HTTP 404.

Impact:
- Telegram/Twitter/Slack/WhatsApp previews will be broken or image-less.
- This is a distribution killer for a public map.

Action:
- Add a real 1200x630 `og-image.png` at the project root/public deploy path, or update meta tags to an existing absolute image.
- Verify with `curl -I https://npz-tactical-map.vercel.app/og-image.png` returning 200.

### 2. AI-agent count is inconsistent

Evidence:
- Homepage livebar says `—Б–Є—Б—В–µ–Љ–∞ –Є–Ј 9 –Ш–Ш-–∞–≥–µ–љ—В–Њ–≤`.
- `/sources` says `6 –Ш–Ш-–∞–≥–µ–љ—В–Њ–≤`.
- Existing docs mention reality around 4-7 routines depending on whether datasets/heartbeats are counted.

Impact:
- OSINT products live on credibility. A user who opens `/sources` immediately sees a mismatch.

Action:
- Pick one truthful taxonomy and use it everywhere.
- Recommended wording if accurate: `6 –Љ–Њ–љ–Є—В–Њ—А–Є–љ–≥–Њ–≤—Л—Е –∞–≥–µ–љ—В–Њ–≤ + 3 —Б–ї—Г–ґ–µ–±–љ—Л—Е –Љ–Њ–і—Г–ї—П`.
- If health is degraded, say so plainly instead of inflating the number.

### 3. `ONLINE 24/7` overpromises realtime accuracy

Evidence:
- Homepage says data updates automatically and is monitored 24/7.
- `/sources` says this is OSINT estimate, no closed/realtime telemetry, updates 4 times per day.
- Project guardrail in `CLAUDE.md`: always show estimate disclaimer.

Impact:
- The current copy can be read as live telemetry, especially near fuel availability and route planning.

Action:
- Replace dense livebar with a trust/status strip:
  `–Ю–¶–Х–Э–Ъ–Р OSINT ¬Ј –Њ–±–љ–Њ–≤–ї–µ–љ–Є–µ 4 —А–∞–Ј–∞ –≤ —Б—Г—В–Ї–Є ¬Ј –њ–Њ—Б–ї–µ–і–љ–µ–µ: 22.06 03:03Z ¬Ј –Є—Б—В–Њ—З–љ–Є–Ї–Є`
- Add degraded/stale state when healthcheck reports dead/stale datasets.

### 4. First-time visitor gets dashboard before framing

Evidence:
- First screen is a dense map/dashboard.
- There is no plain first-screen sentence explaining what this is, who it is for, and what to do first.

Impact:
- Strong visual impression, weak user orientation.

Action:
- Add compact intro/framing copy near the map or as first-load bottom sheet:
  `OSINT-–Ї–∞—А—В–∞ —В–Њ–њ–ї–Є–≤–љ–Њ–≥–Њ –Ї—А–Є–Ј–Є—Б–∞: –≥–і–µ –Э–Я–Ч –њ–Њ–≤—А–µ–ґ–і–µ–љ—Л, –≥–і–µ –µ—Б—В—М –і–µ—Д–Є—Ж–Є—В —В–Њ–њ–ї–Є–≤–∞, –Є –Ї–∞–Ї–Є–µ —А–µ–≥–Є–Њ–љ—Л –њ–Њ–і —А–Є—Б–Ї–Њ–Љ.`
- Add one primary CTA. Pick one: `–Я–Њ–і–њ–Є—Б–∞—В—М—Б—П –љ–∞ —Б–≤–Њ–і–Ї—Г`, `–°–Њ–Њ–±—Й–Є—В—М –Њ –љ–µ—В–Њ—З–љ–Њ—Б—В–Є`, `–°–Ї–∞—З–∞—В—М –і–∞–љ–љ—Л–µ`, or `–Я–Њ–ї—Г—З–∞—В—М –Њ–±–љ–Њ–≤–ї–µ–љ–Є—П –≤ Telegram`.

### 5. Useful UI appears late behind splash/boot

Evidence:
- Browser review saw splash/boot before usable map.
- Technical subagent measured useful UI appearing several seconds after load.
- MiMo also flagged boot progress as not tied to real data loading.

Impact:
- On slow connections the app can feel stuck even if it eventually loads.

Action:
- Render the main shell quickly.
- Shorten or remove artificial boot delay.
- Lazy-init Leaflet/markers after first paint.
- If keeping boot, show real loading states: `map tiles`, `refinery data`, `incident feed`.

## P2 вАФ high-value UX fixes

### 6. Mobile hides analytical context behind unlabeled FABs

Evidence:
- At 390x844, there is no horizontal overflow and bottom nav fits.
- National balance, strike feed, voices, and refinery list are hidden behind icon-only FABs (`рЯУК`, `рЯП≠`).
- MiMo and Codex UX agent both flagged this.

Impact:
- Mobile users see a good map but can miss the analysis that makes the map useful.

Action:
- Add first-load mobile bottom sheet with tabs: `–С–∞–ї–∞–љ—Б`, `–£–і–∞—А—Л`, `–Э–Я–Ч`, `–У–Њ–ї–Њ—Б–∞`.
- Add `aria-label` and visible labels/tooltips for FABs.
- Consider opening `–С–∞–ї–∞–љ—Б` by default on first visit.

### 7. Desktop has too many competing layers

Evidence:
- Left KPI/feed/voices, right N–Я–Ч list, top nav, livebar, timeline, layer toggles, legend, ticker all compete.

Impact:
- New users cannot immediately tell whether the main task is national balance, strikes, refineries, –Р–Ч–°, or forecast.

Action:
- Default to one story layer: `–Ґ–µ–Ї—Г—Й–∞—П —Б–Є—В—Г–∞—Ж–Є—П`.
- Collapse secondary panels until requested.
- Keep feed/list/detail panels available, but not all equally loud on first paint.

### 8. Dense markers overlap in southern Russia / Black Sea area

Evidence:
- Multiple fire/refinery/warning/region/road symbols overlap at common zoom levels.

Impact:
- Users lose spatial confidence.

Action:
- Cluster or spiderfy overlapping events.
- Standardize visual grammar: shape = object type, color = severity/status, outline = selected/current.

### 9. –Р–Ч–° feature needs stronger caveat before route planning

Evidence:
- `/sources` explains station locations are real OSM but fuel availability is regional/network estimate, not pump-level data.
- In the main UI, the caveat is easy to miss.

Impact:
- Users may read the route planner as specific station availability.

Action:
- Near route controls, add:
  `–Ь–∞—А—И—А—Г—В –њ–Њ–Ї–∞–Ј—Л–≤–∞–µ—В —А–Є—Б–Ї –і–µ—Д–Є—Ж–Є—В–∞ –њ–Њ —А–µ–≥–Є–Њ–љ–∞–Љ, –љ–µ –≥–∞—А–∞–љ—В–Є—А—Г–µ—В –љ–∞–ї–Є—З–Є–µ —В–Њ–њ–ї–Є–≤–∞ –љ–∞ –Ї–Њ–љ–Ї—А–µ—В–љ–Њ–є –Р–Ч–°.`

## Technical and accessibility

### 10. Add semantic `<h1>` on root

Evidence:
- Root page has good title/meta but no `<h1>`.

Action:
- Add one visually compatible heading, e.g. `–Ъ–∞—А—В–∞ –Э–Я–Ч –Є –і–µ—Д–Є—Ж–Є—В–∞ —В–Њ–њ–ї–Є–≤–∞`.
- It can be visually integrated with the top brand.

### 11. Improve accessible names and focus behavior

Evidence:
- Many map marker/button-like elements and icon controls lack accessible names.
- Red/orange/green status is heavily color-coded.

Action:
- Add labels for FABs and icon-only controls.
- Remove decorative markers from tab order when they are not meaningful controls.
- Add text/pattern redundancy for refinery status.
- Add pause option for ticker or avoid auto-moving text on mobile.

### 12. Cache first-party static assets better

Evidence:
- `app.js` ~82 KB and `styles.css` ~33 KB are served with `cache-control: public, max-age=0, must-revalidate`.

Action:
- If staying static, use versioned filenames and long cache headers for JS/CSS.
- Keep HTML revalidated.

### 13. Reduce third-party render dependency

Evidence:
- Leaflet and markercluster load from `unpkg.com` as critical scripts.

Action:
- Self-host pinned vendor files or bundle them.
- Add `defer` where safe.
- Keep SRI if external CDN remains.
