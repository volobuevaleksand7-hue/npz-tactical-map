## 4. 🤖 Концепция SEO-агента (только дизайн)

### 4.1. Роль
Автономный агент в пайплайне Hermes (VPS, `claude-haiku-4-5` или `sonnet`), который пишет и затачивает статьи под SEO. Работает как дополнение к существующему `gen-news.py` — расширяет контент, не заменяет сводки.

### 4.2. Источники тем
| Канал | Пример | Частота |
|---|---|---|
| **Wordstat/XMLRiver (платный)** | «почему нет бензина в крыму» (24k/мес) | 2 раза/нед |
| **Google Trends (бесплатно)** | всплески «дефицит бензина + регион» | ежедневно |
| **Новостные поводы** | Путин о дефиците, импорт из Индии, OPEC+ | по событию |
| **Long-tail из gen-news.py** | города/НПЗ с атаками (Кстово — 6 сводок) | авто |
| **SERP-парсинг (Python, бесплатно)** | что Google показывает по «нет бензина» | 1 раз/нед |

### 4.3. Пайплайн

```
wordstat_cluster.py (частотки + ассоциации)
    ↓
agent: «Выбери тему из топа: high-volume + подходит по смыслу»
    ↓
gen-article.py (агент пишет черновик — 500-1200 слов)
    ↓
lint-seo.py (проверка: ключ в H1/title/description/1-й абзац)
    ↓
linx.py (внутренняя перелинковка: ссылки на /news/дат, / , /?view=crimea)
    ↓
schema.py (обёртка NewsArticle / FAQPage / Article)
    ↓
write → news/<slug>.html + обновление sitemap
    ↓
ping (Google Indexing API или sitemap ping)
```

### 4.4. Типы статей (order by impact)

1. **Региональные:** «Почему в Крыму нет бензина — ситуация на 2026» (ловим 38k/мес «в крыму нет бензина»)
2. **По заводам:** «Удары по Рязанскому НПЗ — хроника и последствия» (ловим 2.4k/мес «рязанский нпз атака»)
3. **Объяснялки:** «Почему дефицит бензина в России — 5 причин» (42k/мес «почему нет бензина»)
4. **Сравнительные:** «Дефицит топлива: какие регионы самые проблемные»
5. **Глоссарий/FAQ:** «Что такое талоны на бензин и где они введены» (14k/мес «бензин по талонам»)

### 4.5. Метрики качества

| Метрика | Цель | Как измерять |
|---|---|---|
| Ключ в title | 100% | grep \<title\> |
| Ключ в H1 | 100% | grep \<h1\> |
| Ключ в 1-м абзаце | 100% | head -n1 |
| Внутренние ссылки | ≥3 на статью | grep href= |
| Unique content (дублей нет) | 100% | minhash-дедупликация |
| Readability (FKRA рус) | 40-60 | textstat |
| Word count | 500-1200 | wc -w |

### 4.6. Защита от каннибализации

- **Таблица-регстр:** `data/seo-topics.jsonl` — каждая тема хранит ключи, URL, дату
- Перед генерацией: поиск `seo-topics.jsonl` по ключам-пересечениям → если ≥40% ключей совпадает с существующей статьёй — слияние или отказ
- Для автогенерации: один топик = одна статья. Если через 30 дней новый контент-апдейт — обновить дату, не плодить новую.

### 4.7. Расписание

```cron
# Ежедневно (утром): сбор трендов + предложение темы
0 6 * * *  agents/seo-pick-topic.sh
# После выбора: генерация + публикация
30 7 * * *  agents/seo-gen-article.sh
# Еженедельно: реоптимизация старых статей (title, description, links)
0 10 * * 1  agents/seo-refresh.sh
```

### 4.8. Риски

| Риск | Оценка | Митигация |
|---|---|---|
| **YMYL (Your Money or Your Life)** | Высокий | Дисклеймер OSINT на каждой странице, не даём прямых советов «где купить», ссылаемся на источники |
| **Новостная выдача Google** | Средний | Не все статьи — новости. Explainers и FAQ не попадают в Google News. Можно mix: часть в `/news/`, часть в `/articles/` |
| **Яндекс vs Google** | Низкий | Яндекс чувствительнее к коммерции — у нас нет коммерции, только информация |
| **Дубли контента с источниками** | Средний | Каждая статья — переработка, не копипаст. Добавить canonical на свою версию |
| **Деанонимизация / юридические риски** | Средний | Не публиковать личные данные авторов источника (Telegram). Все цитаты — из открытых каналов |

### 4.9. Приоритет внедрения

1. **Региональная статья по Крыму** — самый высокий объём (86k+38k/мес) — сделать руками или агентом **первой**
2. **Статья «Почему нет бензина в России»** — 109k/мес — обновить index.html и сделать landing
3. **Статья по Рязанскому НПЗ** — 2.4k/мес, но высокий CTR (информационный запрос)
4. **Создать индекс занятых тем** (`seo-topics.jsonl`) — техническая база для защиты от каннибализации

## 5. 🗺️ 3 кандидата в «10 улучшений карты и бота» с точки зрения SEO/трафика

### Кандидат 1. Посадочные страницы по регионам и НПЗ

**Суть:** `/rambler` → статическая страница с картой одного региона (Крым) + пояснительный текст + FAQ + перелинковка на главную и даты.

**Почему SEO:** «топливо в Крыму» — 87k/мес, «почему нет бензина в крыму» — 24k/мес, «бензин по талонам Крым» — 5k/мес. Сейчас эти запросы НЕ имеют целевой страницы на сайте. Единственный путь — главная карта, где нет текста.

**Как сделать:**
- `/krym` — сводка по региону: список АЗС, карта Крыма (выделенная), FAQPage schema, NewsArticle под каждый новый пост
- `/nrz/ryazan` — хроника атак по Рязанскому НПЗ, все даты, ссылки на сводки
- Генерация: агент читает `data/strikes.json` → группирует по городу → пишет summary

**Оценка трафика:** +20-40% органического (на эти запросы сейчас сайт вообще не ранжируется).

### Кандидат 2. FAQ-секция на главной + динамическая FAQPage

**Суть:** развернуть FAQ на главной — не 4 вопроса (как сейчас), а 10-15 вопросов с ответами, обновляемые раз в 2-3 дня.

**Почему SEO:** FAQPage — один из самых «дорогих» rich results в Google (занимает 2-3 позиции на мобильной выдаче). 4 текущих вопроса — старт; 15 вопросов × правильно подобранные ключи = massive coverage.

**Что добавить (примеры):**
- «Почему вырос бензин в 2026» — 3k/мес
- «Какие НПЗ полностью остановлены» — 2k/мес
- «Где заправиться в Крыму 2026» — 4k/мес
- «Будет ли дефицит зимой» — сезонный всплеск

**Автоматизация:** агент собирает топ-N вопросов из Wordstat/Google Trends → генерирует ответы из `fuel-state.json` → добавляет блок в index.html и news.html.

### Кандидат 3. Англоязычная версия (EN OSINT)

**Суть:** EN-страницы `/en`, зеркалирующие контент для международной OSINT-аудитории.

**Почему SEO:** «russia oil refinery drone attack map 2026» — 3-6k/мес (EN). Без EN-версии сайт теряет аудиторию Reuters/Bloomberg-журналистов, аналитиков, иностранных OSINT-сообществ.

**Что сделать:**
- `/en` — перевод главной (WebApplication schema + hreflang)
- `/en/news` — машинный перевод/пересказ последних 7-10 сводок
- `hreflang` — `ru` → `/`, `en` → `/en`, `x-default` → `/`

**Осторожно:** OSINT на английском может привлечь внимание западных аналитиков — это хорошо (трафик), но может ускорить запросы на блокировку в РФ. Юридически — информация из открытых источников, не нарушает законы.

**Оценка трафика:** +10-25% к сессиям (зависит от доли EN-аудитории).
