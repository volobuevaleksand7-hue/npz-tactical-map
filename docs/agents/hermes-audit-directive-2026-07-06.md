## P1 — три отставших слоя (завести рутины)
Аудит показал протухание (порог/возраст):
- **grid-status** → `data/grid-state.json` — 1.5d (порог 18ч). Каденс: каждые 6ч.
- **fuel-availability (АЗС)** → `data/fuel-availability.json` — 1.0d (порог 18ч), и у 88
  станций **0% source_url**. Каденс: каждые 4ч, дата ВСЕГДА сегодняшняя. Если данные из
  OSM — проставь в `meta` явную провенанс-пометку (источник=OSM+дата запроса), а событийные
  статусы АЗС подкрепляй ссылкой. Есть готовый помощник `agents/refresh-osm-stations.py`.
- **fuel-voices (голоса/комментарии)** → `data/fuel-voices.json` — 1.6d (порог 24ч).
  Каденс: каждые 8ч. Цитаты — на русском, с гардом `is_ru` (уже в `broadcast.py`), с
  источником.
  🔴 **ВАЖНО — дата голоса = сегодняшний день.** Сейчас в `fuel-voices.json` НЕТ ни одной
  записи с `date=2026-07-06` (последние — 05.07), поэтому на странице сводки за сегодня
  секция «🗣️ Голоса людей» ПУСТАЯ (шаблон `gen-news.py` рендерит её только `if voices`
  за эту дату). Собирай СВЕЖИЕ комментарии людей про удары/очереди на АЗС и ставь им
  `date` текущего дня (не переноси всё на прошлые даты). Тогда голоса появятся и в сводке,
  и в Telegram-дайджесте. Пользователь ждёт именно этого — «комментарии людей насчёт атак».

Публикация каждого — ТОЛЬКО через `bash agents/git-sync.sh "<msg>" "<hb-ключ>"`
(hb-ключи: `grid-status`, `fuel-availability`, `fuel-voices`). Промпты-образцы:
`agents/update-prompt-grid.md`, `-availability.md`, `-voices.md`.

## P2 — качество и стабильность
- **fuel-state source_url = 54%** — дотяни ссылки на события НПЗ до ~100%.
- **Перегрузка VPS:** в момент аудита sshd не доходил до баннера (машина в свопе/IO).
  Разгрузи по своему же плану (Phase 1): `journalctl --vacuum-size=50M`, `npm cache clean`,
  `uv cache clean`, `pip cache purge`, `apt clean`. Ограничь `max_concurrent_sessions`.
- **Повреждение `config.yaml`** (был `.corrupt.20260705-*.bak`): пиши конфиг атомарно
  (temp+rename) и валидируй перед заменой, чтобы не терять его снова.
- Ранее в логах: `telegram.error.TimedOut`, ошибка `skill_manage('')` — добавь ретрай на
  Telegram и не зови `skill_manage` с пустым именем.

## Инвариант (обязателен для всех слоёв)
1 слой = 1 запуск. Агент пишет ТОЛЬКО свой `data/<layer>.json`, git/heartbeats не трогает
руками — только `agents/git-sync.sh`. Каждое событие — реальный `source_url`, без выдумок,
без будущих дат, координаты/мощности НПЗ не менять. Время владельцу — по МСК (UTC+3).
