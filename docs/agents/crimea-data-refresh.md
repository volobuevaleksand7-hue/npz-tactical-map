# Директива Гермесу: рассинхрон данных по Крыму (T5)

**От:** Claude (Mac). **Кому:** Hermes Agent (hermes-vps). **Тип:** DATA-задача — правь
только `data/history-crimea.json`, своими нативными средствами (web_search, Read/Write),
через обычный проход агента `agents/update-prompt-history.md`.

## Проблема

`crimea.summary` и `crimea.restrictions[]` уже актуальны на 8 июля (топливо частично
вернулось в продажу, по завышенным ценам). А `crimea.stations[].status` у ВСЕХ 4 станций
всё ещё `"dry"` с пометкой «21 июня» в `note` — трёхнедельная дыра. Карта показывает
станции как полностью сухие, хотя сводка уже пишет про частичное возвращение топлива.
Плюс `crimea.restrictions[]` сейчас смешанный формат: часть записей — объекты
`{date, text, source_url}`, часть — голые строки (нет даты и источника).

## Что сделать

1. **Актуализировать `crimea.stations[]`.** Для каждой из 4 станций (Симферополь,
   Севастополь, Ялта, Керчь) найди её РЕАЛЬНЫЙ текущий статус (`ok|limited|talon|dry`) —
   поиском/телеграм-каналами того же типа, что уже использует `update-prompt-history.md`
   (`«Крым АЗС <город> бензин <дата>»`, `Crimeanwind`, местные СМИ). Обнови `status` и
   `note` так, чтобы они не противоречили `summary`/`restrictions` за тот же день. Если по
   конкретной станции свежих данных не нашлось — оставь `status` как есть, но обнови `note`
   явной пометкой возраста («данные на 21 июня, свежих подтверждений не найдено»), не
   выдумывай улучшение статуса без источника.
2. **Нормализовать `crimea.restrictions[]`.** Приведи ВСЕ записи к единому виду
   `{"date": "YYYY-MM-DD", "text": "...", "source_url": "https://..."}`. Для записей-строк
   без даты/источника — либо найди дату и первоисточник по содержанию (WebSearch), либо,
   если восстановить нельзя, укажи ближайшую известную дату события и пометь
   `"source_url": ""` (пустая строка, не выдумывать ссылку).
3. Не трогай `talon_system`, `price_market_rub_per_l`, `routes`, `outlook` без реальных
   свежих данных — это отдельные поля прежнего прохода.

## Источники и запрет на выдумывание

Только верифицируемые открытые источники (СМИ, официальные заявления, спутник/FIRMS,
Telegram с прямой ссылкой на пост). **Не сочинять** даты, статусы станций или тексты
ограничений — если данных нет, честно оставляй пометку об отсутствии подтверждения, а не
правдоподобную догадку.

## Приёмка

```bash
python3 agents/validate-crimea.py --today $(date -u +%F)
```

Должно быть `OK crimea data valid (...)` без `FAIL` и без warning про устаревшие данные
(`Данные устарели: ...`). Схему валидатора не менять — она уже принимает и старый формат
(строки в `restrictions[]`), и целевой (`{date,text,source_url}`), но задача — привести
ВСЕ записи ко второму виду.
