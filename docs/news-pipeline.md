# Как обновляются новости «Топливный фронт РФ» — весь конвейер простыми словами

Запускается по фразе Серёги «обнови новости / обнови карту» → скилл `npz-map-refresh`.
Оркестратор — Claude (я). Я **сам данные не собираю** (дорого), а раздаю работу дешёвым
внешним агентам, проверяю их и делаю один безопасный пуш.

---

## Шаг 0. Смотрю, что устарело
`bash scripts/assess.sh` — таблица по каждому слою: когда обновляли, возраст, STALE или ok.
Обновляю только то, что протухло, **плюс всегда удары и АЗС**.

## Шаг 1. Рассылаю агентов по слоям (параллельно, каждый — свой файл)

Каждый агент получает: 3 железных правила (не выдумывать; писать только свой `data/*.json`;
git/heartbeats не трогать) + путь к своей спеке `agents/update-prompt-*.md`.

| Слой (файл) | Кто идёт | Где ищет / что парсит |
|---|---|---|
| **Удары `strikes.json`** (приоритет) | Claude-субагент (родной WebSearch) или Gemini | новости про удары БПЛА/ракет + Telegram-зеркала `t.me/s/exilenova_plus, radarrussiia, astrapress`. Ищет ВСЕ пропущенные даты, не только сутки |
| **АЗС `fuel-availability.json`** (всегда на сегодня) | DeepSeek + Tavily | цены/лимиты/очереди, биржа СПбМТСБ. Ищет через `scripts/tavily.sh` + `t.me/s/nefte_baza, oil_capital` |
| **НПЗ `fuel-state.json`** | DeepSeek + Tavily | статус заводов, % выбытия мощностей |
| **Крым `history-crimea.json`, дороги `roads.json`, сети `grid-state.json`** | DeepSeek или Gemini | по одному агенту на слой (или объединить в одного «NPZ»-агента) |
| **Голоса `fuel-voices.json`** | DeepSeek + Tavily | реальные отзывы людей с заправок (t.me-зеркала) |
| **Аналитика `forecast/economy/capacity`** | DeepSeek или Gemini | прогноз поверх свежего fuel-state |
| **Подтверждение `strike-confirm.json`, watchdog `health.json`** | детерминир. скрипты | `python3 agents/strike-confirm.py` (GDELT+FIRMS), `healthcheck.py` |

Поисковый инструмент агентов — **Tavily** (`scripts/tavily.sh`, чистый JSON без CAPTCHA).
Порядок предпочтения: **DeepSeek+Tavily → Gemini → MiMo → Claude-субагент (фолбэк)**.

## Шаг 2. Проверяю агентов (анти-фейк, критично)
`git diff --stat data/` + выборочно смотрю новые записи: есть ли **реальный `source_url`**,
правдоподобны ли даты, не выдуманы ли города/цифры. Пусто там, где новости точно были
(особенно удары) → переотправляю слой Claude-субагенту. Лучше меньше, но настоящих записей.

## Шаг 3. Собираю обложки сводок (`scripts/build-covers.py`)
Одна обложка = одна дата, **едина для сайта и Telegram**. Для каждой даты по ЛИД-удару:
1. **Парсю реальное фото события** — `og:image` из статьи-первоисточника (`source_url`),
   с гейтом качества (≥600px, ландшафт, не лого/аватар).
2. **Генерю обложку по нему** — `codex exec image_gen` в режиме **img2img** (референс = реальное
   фото; сохраняем композицию, но это новая AI-иллюстрация → без republish-копирайта, без «фото
   как факт»). Нет годного референса → генерю по тексту (город + сцена).
3. **Вшиваю подпись** — `agents/caption_cover.py` (Pillow, Arial Bold): «Город — что произошло» +
   дата, брендовая подложка. Итог → `assets/cover-<date>.png`.

🔴 Бэкенд картинок = image-кредиты Codex (~14 шт. и «out of credits»). Кончились → часть дат на
общей заглушке `og-image.png` (не поломка). Пополнить и до-гнать `--missing`.

## Шаг 4. Публикую данные (`scripts/publish.sh "<msg>"`)
Валидирует JSON, ловит conflict-маркеры, считает heartbeat-ключи, делает **один** безопасный
`git-sync.sh` push. **Data-слои карты live СРАЗУ** — сайт читает `data/*.json` напрямую с GitHub raw.
Внутри publish автоматически: `gen-news.py` (ниже) + Telegram-рассылка `bot/broadcast.py`.

## Шаг 5. Собираю новостной сайт-архив (`agents/gen-news.py`, вызывается из publish)
Из свежих данных строит:
- `news.html` — индекс `/news`: герой (свежая сводка + обложка) + нац.баланс + лента карточек по дням.
- `news/<дата>.html` — отдельная сводка за каждый день (удары+голоса+навигация; для свежей — баланс/АЗС/биржа).
- `data/news-archive.json` — **накопительный архив** (даты не теряются при подрезке rolling-window).
- `sitemap.xml` + cache-busting `?v=<хэш>` на CSS (иначе браузер держит старый CSS → «вырви глаз»).

## Шаг 6. Деплою сайт
`cd ~/Documents/npz-tactical-map && vercel --prod`. Важно: **data-слои карты live сразу** (GitHub raw),
а **статические страницы сайта (`/news`, `index.html`) — только после деплоя**.

## Шаг 7. Telegram (`bot/broadcast.py`, из publish)
Дайджест в канал **@NPZmap** и в личку подписчикам, **с обложкой дня** (той же `assets/cover-<date>.png`,
что на сайте). Только новое с прошлой рассылки. Токен/подписчики — в `~/.npz-bot/` (вне репо).

## Шаг 8. Отчёт Серёге
Что обновил, ключевые события, догнанные даты, кто из агентов что собрал, что не подтвердилось,
напоминание про жёсткий refresh (Cmd/Ctrl+Shift+R).
