# Telegram-бот «Топливный фронт РФ» — бриф для проектирования

## Что уже есть (факты, не менять допущениями)
- **Сайт**: статика (`index.html` + `app.js` + `styles.css` + `data/*.json`), хостинг **Vercel**
  (project `npz-tactical-map`, домен `https://npz-tactical-map.vercel.app/`). cleanUrls.
- **Данные карты**: `data/*.json` в GitHub-репо `sergeyramas/npz-tactical-map`, обновляются
  вручную скиллом `npz-map-refresh` (внешние агенты → `agents/git-sync.sh` push). Сайт читает
  data с GitHub raw / из статики Vercel. Слои: strikes (удары), fuel-availability (АЗС),
  fuel-voices (голоса людей), grid-state (электросети), fuel-state (НПЗ), forecast/economy.
- **Существующая TG-инфра**: приватный бот `8657624787:...` шлёт watchdog-алерты владельцу
  (chat 609952529). Это ПРИВАТНЫЙ канал алертов — НЕ переиспользовать для публичной подписки.
- **Аналитика**: подключён Vercel Web Analytics (`/_vercel/insights/script.js`).
- Подписки/Telegram-CTA на сайте сейчас НЕТ — делаем с нуля.

## Чего хочет владелец (требования)
1. **Telegram-бот**, публичный. Подписка на него доступна **прямо на сайте** карты (кнопка/виджет).
2. **Каждый день после каждого обновления карты** бот рассылает подписчикам дайджест:
   - **новости** (что произошло — новые удары/события за апдейт), КРАТКО;
   - **«кто что говорит»** — 2-3 свежих голоса людей (fuel-voices), кратко;
   - **ссылка на карту**, в т.ч. на вкладку **АЗС**.
3. Тон — сжатый, дайджест, без воды. Только открытые данные, без выдумок (как и вся карта).

## Вопросы, которые должен решить дизайн (продумать, обосновать выбор)
- **Хостинг бота/бэкенда**: Vercel serverless (webhook) vs отдельный воркер vs запуск из
  pipeline обновления карты (после `publish.sh`). Где живёт код рассылки?
- **Хранилище подписчиков** (chat_id + предпочтения): Vercel KV / Upstash Redis / Supabase /
  jsonblob / файл в приватном месте. Privacy: chat_id — перс-данные, НЕ коммитить в публичный repo.
- **Регистрация подписки с сайта**: кнопка → `https://t.me/<bot>?start=...`; `/start` регистрирует
  chat_id. Нужны ли темы/фильтры (только удары / только АЗС / всё)?
- **Триггер рассылки**: как «после каждого обновления карты» узнаётся факт апдейта? (хук в
  `publish.sh` / GitHub Action on push to data/ / вызывается скиллом). Идемпотентность: не слать
  дважды один апдейт; слать только НОВОЕ (diff слоёв с прошлой рассылки).
- **Генерация дайджеста**: как из diff `data/*.json` собрать краткий текст (новые strikes с
  прошлого раза + топ-голоса + сводка АЗС). Формат сообщения (Markdown/HTML, длина, кнопки-ссылки
  «Открыть карту», «Вкладка АЗС»).
- **Анти-спам / лимиты Telegram** (30 msg/sec, ретраи, блокировки-отписки: чистить мёртвые chat_id).
- **Безопасность**: токен бота — в env/секретах, не в repo. Веб-хук с секретным path/секретом.
- **MVP vs полная версия**: что минимально нужно для запуска, что — потом.

## Ограничения проекта
- Данные/тексты — только из открытых источников, нейтрально, без политики и перс-данных людей.
- Пуш данных карты уже решён (`git-sync.sh`) — бот НЕ трогает git данных, он ЧИТАТЕЛЬ diff'а.
- Секреты внешним моделям не отдавать; в repo секретов нет.
