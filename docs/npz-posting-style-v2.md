# Редполитика v2 — «Топливный фронт РФ» (@NPZmap + сайт)

Статус: v2.0 — ревью Codex пройдено 2026-07-07, правки внесены (лимиты caption/split, state model, preflight). Заменяет `npz-posting-style.md`.
Дата: 2026-07-07. Все времена в постах — **МСК** (UTC+3), всегда с пометкой «МСК».
Эталонный макет страницы сводки: `docs/design/news-post-v2.html`.

## 0. Бренд-голос

- **Кто говорит:** хладнокровный OSINT-аналитик топливной войны. Не пропагандист, не новостной агрегатор, не бот.
- **Zero политического языка.** Никаких оценок сторон, эпитетов, злорадства. Только факты, цифры, последствия для топливного рынка.
- **Пирамида:** первым идёт самое важное событие суток, а не хронология и не биржа.
- **Каждая цифра — с источником или пометкой «оценочно».** Неподтверждённое всегда маркируется.
- **Один факт — один раз.** Повторный удар по объекту подаётся как продолжение истории («3-я атака»), не как новое событие.
- Технические детали (номера установок, млн т/год) — во втором слое (раскрывающийся блок), не в лиде.

## 1. Дизайн-система Telegram (parse_mode=HTML)

**Разрешённые теги:** `<b> <i> <u> <s> <code> <a> <blockquote> <blockquote expandable> <tg-spoiler>`.

**Эмодзи-грамматика (фиксированная, ничего другого):**

| Маркер | Смысл |
|---|---|
| 🔴 | Главное событие / критично |
| 💥 | Удары по инфраструктуре |
| 🏭 | НПЗ, переработка |
| ⛽ | АЗС, дефицит |
| ⚡ | Электроэнергетика |
| 📊 | Биржа, цифры |
| 🗣 | Голоса людей (только реальные цитаты) |
| 🚨 | МОЛНИЯ |
| ⚠️ / ✅ | БПЛА-тревога / отбой |
| 🔄 | Обновление сводки |
| ✓ / ~ | Подтверждено / не подтверждено |
| 📍 | Геопункт |
| 👉 | Ссылка на карту |

**Визуальные правила:**
- Разделитель секций — пустая строка. Никаких линий из символов (`———`, `▰▰▰`) — на разных клиентах ломаются.
- Заголовки секций: эмодзи + `<b>…</b>`, одна строка.
- Ссылки — только именованные `<a href>` («карта ударов», «сводка на сайте»), не голые URL.
- Детализация (полный список ударов, технические параметры) — в `<blockquote expandable>`: пост выглядит коротким, глубина по клику.
- Числа с разрядами: `74 250 ₽/т`. Проценты без десятых, если не критично.
- `<blockquote expandable>` — реальный тег Telegram; вкладывать blockquote в blockquote нельзя; его содержимое **входит** в лимит длины сообщения. Пометка `[expandable]` в примерах этого документа — условное обозначение, в реальном посте только тег.
- Никаких произвольных HTML-атрибутов/классов — только allowlist-теги и `href` у `<a>`.

**Лимиты и режим отправки (по ревью Codex):**
- `sendPhoto.caption` — жёсткий максимум **1024** знака после entity-parsing; целевой ≤ **900**.
- Текстовое сообщение — жёсткий максимум **4096**; целевой ≤ **3800**.
- Правило выбора: если сводка ≤1024 — одно сообщение «фото + caption»; иначе **split-mode**: фото без caption + отдельное текстовое сообщение (fallback уже реализован в `hermes/bot/broadcast.py` — зафиксировать как норму).
- Молния ≤ 500, БПЛА-алерт ≤ 200.

**Обязательный preflight перед отправкой:** экранирование `&`, `<`, `>` во всех данных из источников (включая проверку `href`); длина считается entity-aware (после снятия тегов); проверка allowlist-тегов; нет голых URL и пустых блоков.
Тест-кейсы: 912 знаков → влезает в caption; 2 200 → не влезает (split-mode); >4096 → отказ/сокращение.

## 2. УТРЕННЯЯ / ВЕЧЕРНЯЯ СВОДКА (Telegram + сайт, одна сущность)

Выходит 2 раза: **08:00 МСК** и **20:00 МСК**. Прикрепляется обложка (город + подпись). Утренняя закрывает ночь, вечерняя — день; вечерняя = финал дня и страница `/news/<дата>.html`.

### Скелет

```html
🔴 <b>{Заголовок-крючок: главное событие суток, ≤90 знаков}</b>
<i>Вечерняя сводка · {6 июля}, 20:00 МСК · Топливный фронт РФ</i>

{Лид: 2–4 строки. ЧТО произошло, ГДЕ, ПОЧЕМУ это меняет картину.
Статус: ✓ подтверждено {чем} / ~ ожидает подтверждения.}

💥 <b>Удары за сутки: {N} ({M} — по НПЗ и энергетике)</b>
📍 {Город}: {объект} — {краткий статус} ✓
📍 {Город}: {объект} — {статус} ~
📍 {Город}: {объект} — {статус} ✓
<blockquote expandable>{Полный список остальных ударов + технические детали:
установки, мощности, повторность («3-я атака: 8.05 / 28.06 / 6.07»)}</blockquote>

🏭 <b>Переработка:</b> стоит {X} НПЗ · частично {Y} · выбито ~{Z}% мощностей
⛽ <b>АЗС:</b> сухо — {регионы}; талоны/лимиты — {регионы}
📊 <b>Биржа:</b> АИ-95 {цена} ₽/т ({динамика}), АИ-92 {цена} ₽/т
<blockquote expandable>{Детали: экспорт/импорт, розничные цены по регионам,
электроэнергетика ⚡ если есть}</blockquote>

🗣 <i>«{Одна реальная цитата ≤160 знаков}»</i> — {город}

👉 <a href="https://npz-tactical-map.vercel.app">Карта ударов</a> · <a href="https://npz-tactical-map.vercel.app/news/{дата}">Сводка на сайте</a>
```

**Жёсткие правила:**
1. Блоки 🏭/⛽/📊 — ровно по одной строке в видимой части. Всё остальное — в раскрывающийся блок.
2. 🗣 — только настоящая человеческая цитата из fuel-voices. Нет цитаты — блок опускается (не заменять OSINT-рерайтом).
3. Пустые блоки опускаются целиком (⚡ без событий — нет строки).
4. Если днём была МОЛНИЯ — она и есть «главное»: лид ссылается на неё, не пересказывает заново.
5. Никакого reverse-chronology: порядок = важность.

### Пример (реальные данные 06.07)

```
🔴 Омский НПЗ поражён — первый удар по Сибири, 2 500 км от границы
Вечерняя сводка · 6 июля, 20:00 МСК · Топливный фронт РФ

Дроны достигли крупнейшего НПЗ России (22+ млн т/год). Горят установки

крекинг-катализаторов. ✓ Подтверждено: NASA FIRMS, власти Омска.

💥 Удары за сутки: 15 (8 — по НПЗ и энергетике)
📍 Омск: Омский НПЗ — горят две установки ✓
📍 Ярославль: ЯНОС — 3-я атака (8.05 / 28.06 / 6.07) ✓
📍 Белгород: ТЭЦ — ракетный удар ✓
```

## 3. Пополнение сводки в течение дня

- Одна активная сводка на день (утренняя). Бот **редактирует** пост. Правило (не взаимозаменяемо!): caption-режим → `editMessageCaption` (лимит 1024); split-режим → `editMessageText` текстового сообщения (лимит 4096). Если апдейт переполняет caption — однократное переключение в split-mode (фото остаётся, текст уходит отдельным сообщением со ссылкой из caption).
- Состояние дня (state model, хранится ботом): `date`, `media_message_id`, `text_message_id`, `mode` (caption|split), `last_update_msk`, `version`, `site_page`.
- Апдейт — строка сразу под шапкой: `🔄 <b>{14:20 МСК}</b> {одна строка: что добавилось}` (новые — сверху, максимум 3; дальше — только вечерняя сводка).
- TIER1-событие → отдельная МОЛНИЯ + в сводке строка `🔄 14:20 МСК — 🚨 <a href="{ссылка на молнию}">МОЛНИЯ: {заголовок}</a>`.
- На сайте: блок `<div class="digest-updates">` в шапке страницы + `[обновлено 14:20 МСК]` у изменившейся секции.
- Вечерняя сводка публикуется новым постом (не редактированием утренней) и закрывает день.

## 4. МОЛНИЯ (TIER1 — авто; TIER2 — после кнопки-подтверждения)

```html
🚨 <b>МОЛНИЯ · {Заголовок ≤70 знаков}</b>

📍 {Город, область}
🎯 {Объект} — {одна строка: что это и почему важно}
{✓ Подтверждено: {источники} | ~ Ожидает подтверждения}

{Одна строка контекста: масштаб/последствие/повторность.}
👉 <a href="https://npz-tactical-map.vercel.app">Карта</a>
```

Пример:

```
🚨 МОЛНИЯ · Удар по Омскому НПЗ — 2 500 км от границы

📍 Омск, Омская область
🎯 Омский НПЗ — крупнейший в РФ (22+ млн т/год), единственное
производство крекинг-катализаторов
✓ Подтверждено: NASA FIRMS, два независимых канала

Первый удар по НПЗ в Сибири за всю кампанию.
👉 Карта
```

Правила: только TIER1-триггеры (по классификатору newswatch); один объект = одна молния (повтор в тот же день — редактирование молнии, апдейт-строкой); молния «занимает» слот «Главное» в ближайшей сводке.

## 5. БПЛА-АЛЕРТ (подписчики бота, настраиваемый)

```html
⚠️ <b>Опасность БПЛА</b> · {Калужская, Тульская, Брянская обл.}
🕐 с {03:15 МСК} · 📡 {источник}
👉 <a href="https://npz-tactical-map.vercel.app/radar">Радар</a>
```

Отбой (отдельным сообщением, reply на тревогу):

```html
✅ <b>Отбой</b> · {регионы} · 🕐 {06:40 МСК}
```

Правила: ≤200 знаков; без обложки; не дублировать в канал чаще 1 раза в 30 мин по тому же региону (в личку подписчикам — по их интервалу).

## 6. Пост-сводка на сайте (/news/<дата>.html) — спека для макета

Та же информационная модель, что Telegram-сводка, но полная (без сокрытия в цитаты). Тёмная тактическая тема из `styles.css` (палитра `--bg/--surface/--amber/--red/--green/--teal/--gold`, шрифты `--disp/--mono`).

Структура страницы:
1. **Hero:** обложка дня (city+caption), заголовок-крючок, дата + время последнего обновления МСК, бейдж «утренняя/вечерняя/архив».
2. **Лента апдейтов дня** (`🔄 14:20 МСК — …`) — если были.
3. **Главное** — лид-карточка с бейджем подтверждения (✓/~) и ссылкой на молнию.
4. **Удары за сутки** — таблица/карточки: город, объект, тип (🏭/⚡/порт), статус, счётчик повторности («3-я атака»), ссылка-якорь на карту (`/?strike=<id>`).
5. **Панель метрик** — 4 больших числа: выбито % мощностей, НПЗ стоит/частично, регионов с дефицитом, биржа АИ-95 (с дельтой).
6. **График потерь переработки** — большой (аннотации дат ударов), не спарклайн.
7. **АЗС и цены** — регионы по статусам (сухо/талоны/лимит) цветовой кодировкой.
8. **Голоса** — блок цитат (только реальные).
9. **Футер:** источники, дисклеймер OSINT, перелинковка (пред./след. день, /news, карта, радар), schema.org NewsArticle + BreadcrumbList.

Мобильная версия: метрики 2×2, таблица ударов → карточки, обложка ≤40vh.

## 7. Расписание и маршрутизация (целевая)

| Что | Когда (МСК) | Куда |
|---|---|---|
| Утренняя сводка | 08:00 | канал @NPZmap + сайт (создаёт страницу дня) |
| Апдейты сводки | по событиям, ≤3 | редактирование поста + страница дня |
| МОЛНИЯ | немедленно (TIER1) | канал + подписчики + строка в сводке |
| Вечерняя сводка | 20:00 | канал + сайт (финализирует страницу дня) |
| БПЛА-алерты | по радару, каждые 10 мин цикл | подписчики (по настройкам), канал ≤1/30 мин |

Единый ключ дедупа: `{дата}:{тип объекта}:{город}:{объект}` — общий для молний, сводок и сайта.
