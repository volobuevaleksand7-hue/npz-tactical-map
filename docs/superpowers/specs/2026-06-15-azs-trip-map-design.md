# Дизайн: вкладка «⛽ АЗС» — карта заправок, наличие топлива (green→red) и карта поездки

- **Дата:** 2026-06-15
- **Проект:** npz-tactical-map (Топливный фронт РФ)
- **Статус:** design / на ревью
- **Автор сессии:** Claude + Серёга

---

## 1. Цель

Добавить **отдельную вкладку-карту `⛽ АЗС`** (как у Крыма своя), на которой:

1. Показаны **реальные заправки** (координаты + бренд сети) по региону **Центр + Юг + Крым**.
2. Каждая заправка покрашена **зелёный → красный** по наличию бензина (оценка по статусу её сети в её регионе).
3. Привязаны **реальные комментарии людей** (отзывы с Я.Карт/2ГИС/drive2/drom/СМИ).
4. Есть **«карта поездки»**: пресет-коридоры (М4 «Дон», Таврида…) **и** кастомный маршрут A→B — строится через OSRM, вдоль него подсвечиваются заправки + сводка «где заправиться / где очереди / где сухо».

## 2. Главный принцип — дёшево для агента

Данные разделены на «тяжёлое статичное» и «лёгкое живое». **Агент обновляет только лёгкое.**

| Файл | Кто пишет | Частота | Размер | Содержимое |
|---|---|---|---|---|
| `data/azs-stations.json` | **я, разово из OSM** (опц. ручной refresh раз в месяц) | ~никогда | большой, статичный | `id, brand, lat, lon, city, region, addr` |
| `data/fuel-availability.json` | **агент (уже есть)** | 4×/сутки | маленький (~33КБ) | статус сети × регион, лимит, цена, очередь, `level` |
| `data/fuel-voices.json` | **агент (уже есть)** | ~1×/день | маленький (~19КБ) | реальные отзывы людей, гео-привязка |
| `data/azs-routes.json` | **я, статично** | ~никогда | маленький | пресет-коридоры (точки трассы + название) |

Фронт делает **join** на клиенте: `станция(brand, region) → статус из fuel-availability → цвет`. Стоимость агента не зависит от числа заправок.

## 3. Модель данных

### 3.1 `data/azs-stations.json` (новый, статичный)

```json
{
  "meta": {
    "generated_at": "2026-06-15",
    "source": "OpenStreetMap / Overpass API (ODbL)",
    "regions": ["ЦФО", "ЮФО", "Ставропольский край", "Республика Крым", "Севастополь"],
    "count": 0,
    "note": "Координаты и бренды — реальные (OSM). Наличие топлива — оценка по статусу сети в регионе, НЕ по конкретной колонке."
  },
  "stations": [
    {
      "id": "osm-node-123456",
      "brand": "lukoil",            // нормализованный ключ сети (см. 3.4)
      "brand_label": "Лукойл",      // отображаемое имя
      "lat": 45.0331, "lon": 38.9756,
      "city": "Краснодар",
      "region": "Краснодарский край", // совпадает с region из fuel-availability
      "addr": "ул. Северная, 1"       // если есть в OSM, иначе null
    }
  ]
}
```

### 3.2 `data/fuel-availability.json` (расширяем существующий — минимально)

Уже есть: `regions[].networks[] = {name, status, limit_l, note}` + `regions[].level`.
**Добавляем** в каждый network необязательное поле `level` (5-ступенчатая шкала green→red), чтобы цвет станции брался напрямую без эвристики. Если агент его не заполнил — фронт выводит level из `status` (маппинг ниже), затем из `region.level`.

### 3.3 `data/azs-routes.json` (новый, статичный)

```json
{
  "meta": { "note": "Пресет-коридоры для карты поездки. Геометрия — схематичная по ключевым городам." },
  "routes": [
    {
      "id": "m4-don",
      "name": "М4 «Дон» — Москва → Краснодар → Крым",
      "waypoints": [[55.75,37.62],[51.66,39.20],[47.23,39.70],[45.04,38.98],[45.05,34.10]],
      "cities": ["Москва","Воронеж","Ростов-на-Дону","Краснодар","Симферополь"]
    }
  ]
}
```

### 3.4 Нормализация брендов (генерация azs-stations)

OSM `brand`/`operator`/`name` грязные (`ЛУКОЙЛ`, `LUKOIL`, `Лукойл`). Маппинг raw → ключ:

- `lukoil` ← лукойл, lukoil, teboil (Лукойл-сеть)
- `gazpromneft` ← газпромнефть, gazprom neft, газпром нефть
- `gazprom` ← газпром (АГНКС/АЗС)
- `rosneft` ← роснефть, rosneft, бп/bp(ребренд)
- `tatneft` ← татнефть, tatneft
- `bashneft` ← башнефть
- `surgutneftegas` ← сургутнефтегаз
- `atan` ← атан *(Крым)*
- `tes` ← тэс *(Крым)*
- `eka`, `trassa`, `neftmagistral`, `nnk`, `kalina`, … (по мере встречаемости)
- иначе → `other` (серый «нет данных»)

Регион каждой станции проставляется на этапе генерации **point-in-polygon** по `data/russia-regions.geojson` + `crimea-regions.geojson` + `new-territories.geojson` (shapely).

### 3.5 Маппинг статус → уровень (фронт, если у network нет `level`)

| network.status | level | цвет (есть в `AZS_LVL`) |
|---|---|---|
| ok / normal / available | `calm` | #2f9e57 зелёный |
| minor / some | `strained` | #9fc63a |
| limited / talony | `limited` | #e8b020 жёлтый |
| severe / shortage | `severe` | #e07a18 |
| critical / dry / none | `critical` | #d23a2e красный |
| unknown / null | → `region.level`, иначе серый `#7a7e85` | |

Безымянные станции (`brand=other`) — всегда серые «нет данных», но видны на карте (полезно для поездки).

## 4. Фронт: новая вкладка `view-azs`

- Кнопка таба `⛽ АЗС` в `nav.tabs` сразу после `КРЫМ`.
- `section.view#view-azs` с `#mapAzs` (своя Leaflet-карта, как `mapCrimea`); инициализация — лениво при первом открытии таба.
- **Кластеризация:** Leaflet.markercluster (unpkg, как Leaflet) — точек много, при отдалении схлопываем.
- **Цвет станции:** через join (3.5). Иконка — переиспользуем стиль `azs-pin` (SVG-капля), цвет по уровню.
- **Попап станции:** бренд, адрес, статус сети, лимит л/ТС, цена АИ-95/ДТ, очередь (ч), 1–2 ближайших отзыва + ссылка на источник.
- **Левая управляющая карточка** (`card-left`):
  - Фильтр по **сети** (чекбоксы брендов) и по **уровню** (green…red).
  - Блок **«Карта поездки»** (см. §5).
  - Легенда green→red + дисклеймер (§7).
- **Лента «🗣 КОММЕНТАРИИ»**: список отзывов (из `fuel-voices`), при активном маршруте — фильтруется на города вдоль маршрута.
- **Слой комментариев на карте:** пины-«речи» по городам/станциям; клик → цитата + источник.

## 5. Карта поездки

### 5.1 Пресет-коридоры
Кнопки из `azs-routes.json`. Клик → рисуем polyline коридора, центрируем карту, подсвечиваем заправки в буфере **±5 км (по умолчанию, настраивается)**, показываем сводку (§5.3).

### 5.2 Кастомный A→B (OSRM, клиентский, без ключа)
- Два инпута «Откуда» / «Куда».
- Геокодинг — **Nominatim** (`nominatim.openstreetmap.org/search`, без ключа; уважаем rate-limit, debounce).
- Маршрут — **OSRM demo** (`router.project-osrm.org/route/v1/driving/...?overview=full&geometries=geojson`, без ключа).
- Рисуем геометрию маршрута, находим станции в буфере **±5 км** (ручной haversine-to-segment; turf.js опц.), сортируем вдоль маршрута.

### 5.3 Сводка поездки
- Счётчики вдоль маршрута: 🟢 штатно / 🟡 лимиты / 🔴 сухо / ⚪ нет данных.
- **«Самый длинный красный разрыв»** — макс. расстояние между станциями со статусом ≥ severe (где реально негде заправиться).
- Рекомендованные точки дозаправки (зелёные/жёлтые с минимальной очередью).
- Дисклеймер: оценка, не гарантия.

## 6. Обновление агентом

- **Новый агент не нужен.** Расширяем существующую рутину `availability` (+ `voices`): она уже гоняет WebSearch и пишет `fuel-availability.json` / `fuel-voices.json`. Корректируем её промпт, чтобы заполняла `networks[].level` и привязывала отзывы к городам Центр+Юг+Крым.
- `azs-stations.json` — статичный; опционально низкочастотный (раз в месяц) refresh-скрипт `agents/refresh-osm-stations.py` (Overpass), но в рутину НЕ ставим (тяжёлый, редкий, ручной).
- Фронт уже читает все `data/*.json` из GitHub raw каждые 5 мин → правки агента видны без редеплоя.

## 7. Честность / дисклеймеры

- Бейджи `data_mode`: станции — «OSM · реальное расположение»; наличие — «ОЦЕНКА по сети/региону»; отзывы — «реальные, не репрезентативно».
- Явная плашка на вкладке: **«Расположение заправок — реальное (OpenStreetMap). Наличие топлива — оценка по статусу сети в регионе из открытых источников, не данные по конкретной колонке.»**
- OSM-атрибуция (ODbL) в попапе/футере вкладки.

## 8. Non-goals (YAGNI)

- ❌ Живые данные по конкретной колонке (нет источника).
- ❌ Пошаговая навигация / голос.
- ❌ Аккаунты, сохранение маршрутов.
- ❌ Вся РФ за пределами Центр+Юг+Крым (в этой итерации).
- ❌ Платные API (Google/2ГИС/Яндекс ключи) — только бесплатные OSM/OSRM/Nominatim.

## 9. Этапы реализации (вход для writing-plans)

1. **OSM-генерация** `agents/refresh-osm-stations.py` → `data/azs-stations.json` (Overpass bbox Центр+Юг+Крым → фильтр бренда → нормализация → PIP-регион → запись). Валидация: count>0, все регионы покрыты, нет NaN-координат.
2. **`data/azs-routes.json`** — 4–6 пресет-коридоров.
3. **Фронт-каркас:** таб + `view-azs` + `mapAzs` + ленивая инициализация + markercluster.
4. **Рендер станций** + join green→red + попапы + фильтры по сети/уровню.
5. **Комментарии:** слой на карте + лента + привязка.
6. **Поездка:** пресеты → кастом A→B (Nominatim+OSRM) → буфер станций → сводка.
7. **Расширить рутину агента** (availability+voices): `networks[].level`, отзывы по новым городам. Обновить `sources.html`.
8. **Verify** (preview: консоль чистая, станции рендерятся, маршрут строится, попапы с отзывами) → git push → проверить raw-CDN → обновить auto-memory.

## 10. Риски и смягчение

- **Размер `azs-stations.json`:** фильтр по бренду + (опц.) лимит безымянных + минификация координат до 4 знаков. Цель < ~1.5 МБ (как russia-regions).
- **OSRM/Nominatim rate-limit:** debounce, кэш последнего запроса, фолбэк на пресеты при ошибке.
- **Пробелы в брендах OSM:** часть станций уедет в `other` (серые) — это честно («нет данных»), не ломает карту.
- **Сопоставление region (OSM) ↔ region (availability):** общий нормализатор имён (переиспользовать `normRegion()` из app.js, добавить алиасы).
- **Производительность:** кластеризация + рендер только при активном табе.
